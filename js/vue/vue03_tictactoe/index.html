Ce que fait cette POC = un tictactoe avec vue.js :

  • un module engine.js qui gère un board tictactoe et ses fonctions utilitaires (qui gagnerait à être une classe d'ailleurs)
  • un composant vue parent App, qui n'est qu'un wrapper autour d'un composant Tictactoe (lui-même utilisant un sous-composant Cell )
  • le composant Tictactoe est une IHM pour utiliser engine.js pour faire des parties
  • l'app indique le gagnant (via une computed property réagissant aux modifications du board), et on peut reset vers une nouvelle partie à tout moment
  • utilisé : les principes généraux vue.js (composants, sous-composants, template, utilisant de modules js indépendants de vue, events, props, etc.), modules js, classes js, ...
